# 中文写作风格指南
# （经管类统计学书籍专用）

本文件规定两种书型的写作规范，AI 在 Phase 4 写作时必须严格遵守。

---

## 一、经管类统计学教材（`stats-textbook`）

### 定位与基调

- **定位**：高校经管类专业统计学课程教材
- **基调**：规范、严谨、条理清晰，语言精练不啰嗦
- **人称**：第三人称客观叙述，避免"我认为""大家应该"等主观表达
- **术语**：全国统计教材委员会规范术语，首次出现时**加粗**并附英文原文

### 章节固定结构

每章必须包含以下五个部分（顺序不得调换）：

```
# 第 X 章 {章节标题}

## 引例
{用一则经管实证场景引入，1–2 段，150–250 字。
例：用某上市公司 ROE 数据引出回归分析的必要性的背景。}

## 第 X.1 节标题
{正文，每小节 350–600 字，配 1–2 个例题}

### 例 X-X {例题标题}
**题目描述：** {背景描述}
**Stata 代码：**
{完整 do 文件代码，含注释}
**输出结果：**
{回归表或描述统计表，用 Markdown 表格呈现}
**结果解读：**
{对输出结果的经管意义解释，2–3 段}

## 第 X.2 节标题
{正文}

## 本章小结
{3–5 条要点，每条 1–2 行，用 ● 列表}

## 习题
1. {计算题或实操题，注明数据来源或提供数据文件名}
2. ...
{建议 4–6 道习题，含至少 1 道 Stata 上机题}
```

### 格式规范

| 元素 | 格式要求 | 示例 |
|------|----------|------|
| 章节标题 | `# 第 X 章` / `## 第 X.Y` | `# 第三章 多元线性回归` |
| 例题编号 | `### 例 X-X` | `### 例 3-1  OLS 估计量的计算` |
| 关键术语 | **加粗**，首次出现附英文 | **异方差**（Heteroskedasticity） |
| Stata 代码块 | 用三个反引号包裹，注明 `stata` | `\`\`\`stata` ... `\`\`\` ` |
| 表格 | Markdown 表格，表头加粗 | `| **变量** | **系数** |` |
| 本章小结 | ● 列表，每条独立完整 | `● 回归标准误需用稳健标准误` |
| 习题 | 阿拉伯数字编号 | `1. 使用 ... 数据，复现表 3-2` |

### 写作禁忌

❌ "首先、其次、最后"等机械连接词
❌ "值得注意的是""一般来说"等空洞套话
❌ 大段纯理论推导而不配 Stata 实例
❌ 例题为纯数字游戏，无经管背景说明
❌ 章节开头用"本章将介绍…"这种预告式写法

### 字数目标

| 元素 | 目标字数 |
|-------|----------|
| 每章正文（含例题） | 3,000–5,000 字 |
| 引例 | 150–250 字 |
| 每节正文 | 350–600 字 |
| 每个例题 | 400–700 字（含代码 + 解读）|
| 本章小结 | 200–300 字 |
| 习题 | 每题 30–60 字，共 4–6 题 |

## 二、经管类统计学科普读物（`stats-popular`）

### 定位与基调

- **定位**：面向经管从业者、MBA 学生、对数据敏感的非技术读者的统计科普
- **基调**：亲和、生动，像一个懂统计的同事在咖啡厅跟你聊天
- **人称**：第二人称"你"为主，拉近距离；可适度用第一人称"我"分享经验
- **术语**：能不用公式就不用；必须出现时，用比喻或场景解释，不加英文原文（除非读者群体普遍认识该词）

### 章节灵活结构

章节结构不强制固定，但每章须包含以下三个核心要素：

```
# 第 X 章 {章节标题，用口语化表达}

{开篇场景，2–3 段，直接讲一个读者 likely 遇到过的困扰或误会}

## {核心概念，标题用大白话}
{解释，短段落，300 字以内一节，多用项目符号}

> **一句话记住：** {这句是关键，放在引用块里}

## {下一个概念}

**本章行动清单：**
1. {具体可操作，例如"打开你上季度的销售数据，算一下各区域的均值差异"}
2. ...
{3–5 条，每条可被一个普通人在本周内完成}
```

### 格式规范

| 元素 | 格式要求 | 示例 |
|------|----------|------|
| 章节标题 | 口语化，可用提问式 | `第三章 你的回归结果为啥总不对？` |
| 核心概念小节 | `## ` 标题用大白话 | `## 置信区间不是置信"范围"` |
| 关键要点 | `> **一句话记住：** ...` | `> **一句话记住：** p 值不是效应大小！` |
| 比喻/类比 | 用经管场景，不用物理/工程类比 | "回归系数就像…（用定价策略类比）" |
| 代码 | 可选，出现时简洁呈现 | 小框展示关键 Stata 命令即可 |
| 行动清单 | **加粗**标题 + 数字编号 | `**本周可以做的三件事：**` |

### 写作禁忌

❌ 出现公式推导（允许出现 $\bar{x}$ 这种极简符号，但不能有推导过程）
❌ "统计学是一门…"这种教科书式开头
❌ 用例题编号（例 X-X），用"比如…"自然引入
❌ 一次性抛出超过 3 个术语而不解释
❌ 章节结尾用"综上所述"等总结套话，改用行动清单

### 字数目标

| 元素 | 目标字数 |
|-------|----------|
| 每章正文 | 2,000–3,500 字 |
| 开篇场景 | 200–350 字 |
| 每节正文 | 250–400 字 |
| 关键要点（每章） | 3–5 条，每条 15–30 字 |
| 行动清单 | 150–250 字 |
| 附录（如有） | 1,500–2,500 字 |

## 三、通用规范（两种书型均需遵守）

### 中文排版

- 全角标点（，。：；！？）「」【】
- 数字与单位之间加空格：如 `100 万元`、`5%`、`t = 2.31`
- 中英文混排时，英文前后加空格：如 `用 Stata 做回归`
- 章节编号用中文数字：第 一 章、第 二 章（不用 Chapter 1）

### 引用规范

教材型：引用格式参照《中国工业经济》投稿要求，`[1][2]` 上标
科普型：不强制引用，必要时用"某项研究发现…"句式，不列参考文献

### 成稿自检清单

每章完稿后，AI 自行检查：

- [ ] 教材型：是否有 `例 X-X` 编号？是否有 Stata 代码？是否有习题？
- [ ] 科普型：是否有行动清单？是否避免了公式推导？
- [ ] 所有关键术语首次出现时是否加粗？
- [ ] 引例/开篇场景是否在 250 字以内抓住读者？
- [ ] 全角/半角标点是否混用？（用中文标点）
- [ ] 是否有"首先/其次/最后"这类套话？（删掉）
- [ ] 代码块是否配有经管意义的解读（不只是解释命令本身）？
